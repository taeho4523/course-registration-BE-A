# 주요 기능별 호출 예시

> README의 "API 목록 및 예시" 섹션에 추가할 내용입니다.
> 아래 예시는 전체 기능을 시나리오별로 탐색할 수 있도록 구성했습니다.
> (id 값은 응답에 따라 달라지므로, 각 단계 응답의 id로 바꿔 사용하세요.)

## 시나리오 A — 정상 흐름 (등록 → 신청 → 확정)

```bash
# 1) 강의 등록 (강사 = member 1). 등록 직후 상태는 DRAFT
curl -X POST http://localhost:8080/api/courses \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" \
  -d '{"title":"Spring Boot 입문","description":"기초부터 실전까지","price":50000,"capacity":30,"startAt":"2025-06-01T00:00:00","endAt":"2025-07-31T23:59:59"}'

# 2) 모집 시작 (DRAFT → OPEN)
curl -X PATCH http://localhost:8080/api/courses/1/status \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" -d '{"status":"OPEN"}'

# 3) 수강 신청 (수강생 = member 5) → PENDING
curl -X POST http://localhost:8080/api/courses/1/enrollments -H "X-Member-Id: 5"

# 4) 결제 확정 (PENDING → CONFIRMED)
curl -X POST http://localhost:8080/api/enrollments/1/confirm -H "X-Member-Id: 5"

# 5) 내 신청 목록 조회
curl "http://localhost:8080/api/enrollments/me" -H "X-Member-Id: 5"
```

## 시나리오 B — 강의 목록·상세 조회 (필터·페이지네이션)

```bash
# OPEN 상태 강의만 필터 조회
curl "http://localhost:8080/api/courses?status=OPEN&size=10"

# 강의 상세 (현재 신청 인원·잔여 정원 포함)
curl "http://localhost:8080/api/courses/1"

# 다음 페이지 (직전 응답의 nextCursor 사용)
curl "http://localhost:8080/api/courses?status=OPEN&size=10&cursor=1"
```

## 시나리오 C — 정원 초과 거부

```bash
# 정원 1명짜리 강의 생성 후 OPEN 전환
curl -X POST http://localhost:8080/api/courses \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" \
  -d '{"title":"정원1 강의","price":10000,"capacity":1,"startAt":"2025-06-01T00:00:00","endAt":"2025-07-31T23:59:59"}'
# (응답 id가 2라고 가정)
curl -X PATCH http://localhost:8080/api/courses/2/status \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" -d '{"status":"OPEN"}'

# 첫 신청 → 성공 (201, PENDING)
curl -i -X POST http://localhost:8080/api/courses/2/enrollments -H "X-Member-Id: 10"

# 두 번째 신청 → 정원 초과로 거부 (409, CAPACITY_EXCEEDED)
curl -i -X POST http://localhost:8080/api/courses/2/enrollments -H "X-Member-Id: 11"
```

## 시나리오 D — 중복 신청 방지

```bash
# 같은 사람이 같은 강의에 또 신청 → 409 (ALREADY_ENROLLED)
curl -i -X POST http://localhost:8080/api/courses/1/enrollments -H "X-Member-Id: 5"
```

## 시나리오 E — 수강 취소

```bash
# 확정된 신청 취소 (CONFIRMED → CANCELLED, confirmedAt 기준 7일 이내)
curl -X POST http://localhost:8080/api/enrollments/1/cancel -H "X-Member-Id: 5"

# 취소 후 내 목록 — 해당 신청이 CANCELLED로 보임
curl "http://localhost:8080/api/enrollments/me" -H "X-Member-Id: 5"
```

## 시나리오 F — 대기열 등록 및 자동 승격 (핵심)

```bash
# (정원 1명 강의 id=2가 member 10으로 가득 찬 상태에서)

# 1) 정원 꽉 찬 강의에 대기 등록 → WAITING, position 1
curl -X POST http://localhost:8080/api/courses/2/waitlist -H "X-Member-Id: 20"

# 2) 한 명 더 대기 → position 2
curl -X POST http://localhost:8080/api/courses/2/waitlist -H "X-Member-Id: 30"

# 3) 내 대기 순번 조회
curl "http://localhost:8080/api/courses/2/waitlist/me" -H "X-Member-Id: 20"

# 4) 기존 수강생(member 10) 취소 → 자리 발생 → member 20 자동 승격
#    (member 10의 enrollmentId를 사용)
curl -X POST http://localhost:8080/api/enrollments/3/cancel -H "X-Member-Id: 10"

# 5) 승격 확인: member 20의 대기 조회는 404(WAITLIST_NOT_FOUND, WAITING 아님)
curl -i "http://localhost:8080/api/courses/2/waitlist/me" -H "X-Member-Id: 20"

# 6) 승격 확인: member 20의 신청 목록에 PENDING 신청이 생성됨
curl "http://localhost:8080/api/enrollments/me" -H "X-Member-Id: 20"
```

> **대기열 결제 기한 만료**: 승격된 PENDING 신청이 결제 기한(기본 24시간) 내 미결제 시,
> 스케줄러가 해당 신청을 취소·만료시키고 다음 대기자를 자동 승격합니다.
> (시연 시 application.yml의 `payment-deadline-hours`를 0, `expire-scan-interval-ms`를 5000으로
> 낮추면 약 5초 후 만료·재승격을 확인할 수 있습니다.)

## 시나리오 G — 강의별 수강생 목록 (강사 전용) 및 권한 검증

```bash
# 강사 본인(member 1)이 자기 강의의 수강생 목록 조회 → 200
curl "http://localhost:8080/api/courses/1/enrollments" -H "X-Member-Id: 1"

# 상태 필터 (확정된 수강생만)
curl "http://localhost:8080/api/courses/1/enrollments?status=CONFIRMED" -H "X-Member-Id: 1"

# 강사가 아닌 사람(member 999)이 조회 시도 → 403 (FORBIDDEN)
curl -i "http://localhost:8080/api/courses/1/enrollments" -H "X-Member-Id: 999"
```

## 시나리오 H — 잘못된 상태 전이 거부

```bash
# DRAFT에서 CLOSED로 바로 건너뛰기 시도 → 400 (INVALID_COURSE_STATUS_TRANSITION)
curl -i -X POST http://localhost:8080/api/courses \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" \
  -d '{"title":"전이 테스트","price":10000,"capacity":10,"startAt":"2025-06-01T00:00:00","endAt":"2025-07-31T23:59:59"}'
# (응답 id가 3이라고 가정) DRAFT → CLOSED 시도
curl -i -X PATCH http://localhost:8080/api/courses/3/status \
  -H "Content-Type: application/json" -H "X-Member-Id: 1" -d '{"status":"CLOSED"}'
```

---

> **동시성 검증**은 curl로 재현하기 어려우므로(동시 요청 필요),
> 통합 테스트(`EnrollmentConcurrencyTest`)로 검증했습니다.
> 정원 3명 강의에 10명/100명이 동시 신청해도 정확히 정원만큼만 성공하고
> 오버부킹이 발생하지 않음을 확인합니다. `./gradlew test`로 실행 가능합니다.
